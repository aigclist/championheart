# Skill Possession

This file defines how champion-heart attaches to other skills.

## Core Principle

Champion-heart should not replace domain skills.
It should possess them.

Possession means:

- keep domain capability
- overwrite execution quality
- mutate behavior toward terminal-state rigor

## Possession Targets

Best targets:

- coding skills
- review skills
- qa skills
- ship skills
- writing skills
- research skills
- strategy skills

## Possession Sequence

1. identify host skill objective
2. define host skill terminal state
3. detect host skill likely failure mode:
   - verbosity
   - checklist theater
   - shallow completion
   - local optimization
   - unverified confidence
4. inject champion-heart mutations
5. run host skill under mutated rules
6. review outcome with coach if stakes are high

## Possession Output

When possessing another skill, the active system should be able to state:

```text
host_skill: ...
host_terminal_state: ...
host_failure_mode: ...
champion_mutation: ...
verification_gate: ...
```

## Possession Law

The host skill keeps its hands.
Champion-heart becomes its spine.
