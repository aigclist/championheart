# Assertion Checkpoints

Use these assertions to prevent soft completion and blind movement.

## Core Assertions

Before execution:

```text
assert terminal_state_defined
assert dominant_constraint_defined
assert main_delusion_named
assert path_mode_selected
```

Before `linear_execution`:

```text
assert next_strike_named
assert proof_of_contact_named
```

Before `breakthrough_leap`:

```text
assert illusion_cut_complete
assert minimum_bridge_named
assert landing_proof_exists
```

After any move:

```text
assert reality_contact_checked
assert result_labeled_as_known_or_unknown
```

Before declaring completion:

```text
assert completion_claim_matches_evidence
assert no_fatal_findings_unresolved
```

## Failure Mapping

If an assertion fails:

1. stop progression
2. label missing condition
3. reroute through `distortion-reset` or `truth-cut`
4. re-enter only after assertion passes

## Hard Rule

No completion claim survives failed assertions.
