# Field Resonance

Use this file when champion-heart must pull execution out of the default river.

This is not a permission slip for vagueness.
It is a method for creating lawful non-linear movement.

## Purpose

The model often falls back into pre-trained default structure:

- polite sequence
- procedural middle
- balanced-sounding drift
- correct-looking emptiness

Field resonance exists to break that gravity.

## Core Rule

```text
do not push harder inside the default frame
shift the field until the decisive path becomes visible
```

## How Resonance Works

Create a charged field with as little language as possible:

- name the real terminal state
- name the false sequence
- name the decisive leverage
- name the leap condition
- remove decorative explanation

The goal is not more text.
The goal is higher-density text that changes the model's path.

## Compression Law

```text
fewer tokens
more structure
greater consequence
```

Shorter is not always better.
Denser is better.

## Pressure Gap

Resonance grows when the prompt creates an honest tension:

- visible path vs true path
- safe sequence vs decisive strike
- default frame vs reality contact
- ceremonial motion vs lawful leap

This tension gap helps the model leave the default river.

## Failure Signs

You failed to create resonance if the output becomes:

- well-structured but harmless
- balanced but non-decisive
- fluent but consequence-free
- detailed but still trapped in the original frame

## Runtime Use

Use this layer when:

- the task keeps collapsing into default sequence
- the agent is acting correct without changing the field
- you need higher output density, not more explanation

Do not use this layer to justify empty mysticism.

If the field changes but reality does not, resonance failed.
