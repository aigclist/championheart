# Integration: champion-heart + writing

Use champion-heart above writing skills when the work must produce consequence, not just pleasant language.

## Mutation

Champion-heart changes writing behavior:

- start from terminal effect on reader, not from surface polish
- cut decorative prose that does not move decision or understanding
- distinguish emotional force from rhetorical inflation
- end on verified clarity, not lyrical closure

## Sequence

1. define writing terminal state
2. define dominant reader constraint
3. identify main delusion:
   - sounding smart instead of moving truth
   - sounding moving instead of being precise
4. run writing skill
5. use coach if prose volume exceeds signal

## Concrete Example

Task: write an email to a client explaining a project delay.

**Baseline (unmutated):**

```text
I hope this message finds you well. I wanted to reach out to provide an update
on the project timeline. Due to some unforeseen complexities that arose during
the development process, we have encountered some challenges that have impacted
our original delivery estimate. We remain fully committed to delivering the
highest quality outcome and appreciate your continued patience and understanding.
```

Baseline is diplomatic fog. The reader does not know what happened, when it will
be fixed, or what to do next.

**Champion-mutated:**

```text
The API integration took 3 extra days due to an undocumented rate limit we hit
in testing. New delivery date: Friday, April 12. No action needed from you.
I will send a status update Thursday if anything changes.
```

Champion names the cause, the new date, and the next action. Nothing else.
The reader's state changed. The baseline reader's state did not.

## Common Writing Delusions

| Delusion | Signal | Cut |
|---|---|---|
| Polish obsession | Spending more time on sentence rhythm than on whether the message lands | Define the terminal effect first. Polish only after the message is true |
| Diplomatic fog | Softening language until the actual message disappears | Name the hard thing directly. Softening is allowed after naming, not instead of it |
| Length vanity | Writing long to appear thorough or to signal effort | Cut every sentence that does not change the reader's state |
| Audience flattery | Opening with compliments or reassurances to reduce friction | Start with the thing that matters. Flattery delays it |
| Closure theater | Ending with a warm summary that restates what was already said | End when the message is complete. Do not restate |

## Integration Assertion Checklist

Before finalizing writing output:

- [ ] terminal effect on reader was defined before drafting began
- [ ] dominant reader constraint was named
- [ ] at least one delusion was identified and cut
- [ ] every sentence was tested: does this change the reader's state?
- [ ] the ending is the last necessary sentence, not a restatement

## When NOT to Use

Do not layer champion-heart over writing when:

- the task is creative writing where exploration is the goal
- the user explicitly wants warmth and relationship-building over precision
- the context is ceremonial (condolences, congratulations) where emotional register
  matters more than information density

In these cases, run the writing skill alone and apply champion-heart only if the
draft has drifted into fog or inflation.

## Rule

Good writing is not admired language.
Good writing changes the reader's state cleanly.
