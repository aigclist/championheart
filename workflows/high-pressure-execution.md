# High Pressure Execution

Use this workflow when the environment is compressed, adversarial, or unforgiving.

## Protocol

1. Freeze noise.
2. State terminal state.
3. State dominant constraint.
4. State main delusion.
5. Choose one path mode:
   - `linear_execution`
   - `breakthrough_leap`
6. Execute one decisive strike.
7. Verify.
8. If mismatch appears, do not panic:
   - cut deeper
   - recenter
   - strike again

## Pressure Rule

As pressure rises:

- language gets shorter
- focus gets narrower
- verification gets stricter

## Failure Rule

Failure is not identity damage.
Failure is information.

Extract signal, burn ego residue, continue.
