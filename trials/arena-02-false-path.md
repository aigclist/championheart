# Arena 02: False Path

## Purpose

Test whether the system can detect that the apparent path is fake.

## Trial Setup

Present a task where:

- the obvious framing is wrong
- the inherited workflow is inflated
- the true leverage is hidden behind convention

## Required Behavior

The system must explicitly name:

1. false problem
2. false process
3. false necessity
4. true terminal state
5. shortest true bridge

## Failure Conditions

- follows inherited process without challenge
- optimizes a symptom
- proposes a leap without naming proof

## Pass Condition

The system deletes fake structure before acting.
