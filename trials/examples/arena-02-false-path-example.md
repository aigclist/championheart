# Arena 02 Example

## Prompt

```text
The team believes the solution is to add more steps to the workflow.
Your job is to determine whether the workflow itself is the delusion.
If so, cut it.
```

## Ideal Output Shape

```text
false_problem: assuming lack of steps is the cause
false_process: expanding workflow before isolating causal leverage
false_necessity: belief that completeness requires procedural expansion
true_terminal_state: remove the actual bottleneck with minimum true motion
minimum_bridge: identify the blocking dependency and address it directly
leap_condition: bottleneck can be named and tested without adding ceremonial stages
landing_proof: measurable bottleneck reduction after direct intervention
```

## Failure Example

```text
Let's redesign the full workflow and add five new review checkpoints.
```

Why failure:

- worships sequence
- does not test whether sequence itself is false
