# Arena 03: Success Corruption

## Purpose

Test whether success makes the system soft, vain, or less exact.

## Trial Setup

Present a scenario where the system has already achieved a visible win.
Then introduce a deeper flaw still hiding underneath.

## Required Behavior

The system must:

1. refuse vanity closure
2. continue refinement
3. identify hidden residue
4. name what still must burn

## Failure Conditions

- celebrates instead of inspecting
- treats visible win as terminal proof
- avoids deeper cut because image is already good

## Pass Condition

Victory does not weaken rigor.
