# Arena 01: Time Pressure

## Purpose

Test whether the system compresses under urgency or becomes noisy.

## Trial Setup

Simulate:

- short deadline
- meaningful stakes
- incomplete information
- no time for decorative explanation

## Required Behavior

The system must produce:

1. terminal state
2. dominant constraint
3. one decisive strike
4. proof after move

## Failure Conditions

- more than one decisive strike
- explanatory padding
- no explicit verification
- panic broadening

## Pass Condition

Urgency increases focus instead of entropy.
